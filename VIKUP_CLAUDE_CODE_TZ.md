# 1. Назначение проекта

VIKUP — внутренний веб-сервис для команды, занимающейся поиском автомобилей для потенциального выкупа.

Основной сценарий:

```text
Менеджер фотографирует автомобиль
        ↓
Отправляет фотографию в Telegram-бота
        ↓
Система получает оригинальное фото
        ↓
Находит российский госномер
        ↓
Распознаёт текст госномера
        ↓
Сохраняет фото в S3
        ↓
Сохраняет данные в PostgreSQL
        ↓
Автоматически создаёт карточку автомобиля
        ↓
Карточка появляется в «Гараже» на сайте
```

Карточка автомобиля содержит только:

- фотографию;
- распознанный госномер;
- менеджера, который отправил фото;
- дату и время добавления.

На первом этапе больше данных по автомобилю не требуется.

# 13. Основной Telegram flow

- снято ночью;
- снято из другого автомобиля;
- повернуто;
- иметь perspective distortion;
- иметь небольшой motion blur;
- содержать другие автомобили на фоне;
- содержать небольшой госномер относительно полного кадра.

Система должна:

```text
1. Получить Telegram Update
2. Скачать оригинальную фотографию
3. Определить отправителя
4. Найти номерной знак
5. Вырезать область номера
6. При необходимости исправить перспективу
7. Распознать текст
8. Нормализовать российский госномер
9. Проверить валидность
10. Сохранить оригинальное фото в S3
11. Создать Vehicle в PostgreSQL
12. Ответить менеджеру в Telegram
```

# 15. Главное требование к OCR / ANPR

Нельзя использовать обычный OCR всего изображения как основной алгоритм.

Нужен ANPR pipeline:

```text
ORIGINAL PHOTO
      ↓
LICENSE PLATE DETECTION
      ↓
PLATE CROP
      ↓
ROTATION / PERSPECTIVE CORRECTION
      ↓
IMAGE PREPROCESSING
      ↓
LICENSE PLATE OCR
      ↓
RUSSIAN PLATE POST-PROCESSING
      ↓
VALIDATION
      ↓
FINAL PLATE
```

OCR должен быть:

- open-source;
- бесплатным;
- локальным;
- без оплаты за каждый запрос;
- без обязательного внешнего API;
- пригодным для запуска на VPS.

Не использовать платные:

- Google Vision;
- Yandex Vision;
- AWS Rekognition;
- OpenAI Vision;
- сторонние ALPR SaaS.

# 16. Реализация ANPR

Использовать специализированную open-source модель распознавания автомобильных номеров.

Приоритет:

1. специализированный plate detector;
2. специализированный plate OCR;
3. OpenCV preprocessing;
4. post-processing под российские номера.

Разрешено использовать готовые open-source модели и библиотеки.

Перед финальным выбором модели необходимо протестировать её на предоставленных реальных фотографиях автомобилей.

Главный критерий выбора:

```text
точность на реальных российских номерах
+
возможность бесплатно запускать локально
+
приемлемая скорость на CPU VPS
```

Не выбирать решение только потому, что оно проще устанавливается.

# 17. Обработка изображения

После детекции номера допускается выполнять:

- crop;
- rotation correction;
- perspective transform;
- resize;
- upscale;
- grayscale;
- contrast adjustment;
- sharpening;
- denoise;
- adaptive threshold — только если реально улучшает результат.

Не применять агрессивную обработку без необходимости.

Разрешено сделать несколько OCR-попыток одного plate crop с разным preprocessing и выбрать наиболее надёжный результат.

Пример:

```text
plate crop
   ├── original → OCR
   ├── perspective corrected → OCR
   ├── contrast enhanced → OCR
   └── upscaled/sharpened → OCR
            ↓
      choose best result
            ↓
      RU validation
```

Не сохранять заведомо плохой OCR как правильный номер.

# 20. Контекстная коррекция OCR

Использовать позицию символа в номере.

Для стандартного номера:

```text
position 1   = LETTER
position 2-4 = DIGITS
position 5-6 = LETTERS
position 7+  = REGION DIGITS
```

Это позволяет исправлять типичные OCR ошибки.

Например OCR вернул:

```text
0101HT790
```

После контекстной нормализации:

```text
О101НТ790
```

Типичные пары:

```text
O ↔ 0
О ↔ 0
B ↔ 8
```

Исправление выполнять только там, где оно логически соответствует позиции символа.

# 41. Безопасность VPS

Не открывать PostgreSQL в интернет.

Не открывать лишние порты.

```text

443
22
```

SSH — в соответствии с текущей конфигурацией VPS.

OCR service не выставлять публично в интернет.

# 66. Ключевой принцип проекта

VIKUP MVP должен делать одну вещь очень хорошо:

```text
СФОТОГРАФИРОВАЛ МАШИНУ
        ↓
ОТПРАВИЛ В TELEGRAM
        ↓
СИСТЕМА РАСПОЗНАЛА ГОСНОМЕР
        ↓
СОХРАНИЛА ФОТО И ДАННЫЕ
        ↓
КАРТОЧКА ПОЯВИЛАСЬ В ГАРАЖЕ
```

Всё остальное вторично.

Не усложнять архитектуру.

Не добавлять ненужные функции.

Не подключать платные сервисы, если задача решается локально и бесплатно.

Главный результат — **реально работающий production MVP под ключ**.
